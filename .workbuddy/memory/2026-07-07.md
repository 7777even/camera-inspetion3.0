## 关键经验沉淀（避免下次再踩）
1. **MCP Java SDK 0.10.0 + Spring AI 1.0.0 行为**：
   - webmvc 仅 SSE server transport（`WebMvcSseServerTransportProvider`）
   - SSE 端点以 functional RouterFunction 注册，路由路径 = sse-endpoint / sse-message-endpoint 本身
   - `base-url` **不前缀到路由**，但**会拼到 endpoint 事件下发的 POST 路径** → base-url 必须为空
   - `MethodToolCallbackProvider` 会**静默丢弃返回 Object/Map 的 @Tool 方法** → 操作类必须返回具体 DTO
2. **C9 测试构造要点**：
   - `HttpClientSseClientTransport.builder(baseUri).sseEndpoint("/mcp/sse").build()` 是正确写法（baseUri 不含 sseEndpoint，SDK 内部 `URI.resolve` 拼出 GET URL）
   - 默认 `sseEndpoint = "/sse"`，不要省略 `.sseEndpoint(...)` 除非确认 baseUri 已含 sse 路径
3. **Java HttpClient SSE 行为**：发到 `java.net.http.HttpClient` 的 SSE 流的 endpoint 事件，由 `HttpClientSseClientTransport$1.onEvent` 处理，设置 `messageEndpoint` 原子引用并 `closeLatch.countDown()`（误导命名，实际是 ready 锁）。`sendMessage` 在调用前 `closeLatch.await(10s)` 等待 ready。
