## 端到端冒烟（@Scheduled + Feishu webhook 真实验证）
- 环境：MySQL 5.7.39（已装）、Python 缺 cv2（无海康摄像头）
- 临时改 cron 到每分钟 + 配飞书 webhook 56caf1a4-29e3-4552-9a8c-3521be2fdc88
- **真实抓到 2 个生产 bug**（unit test 没覆盖）：
  1. **Bug #1**: `InspectionService.findActive(0, 10000)` → MySQL `LIMIT -10000, 10000` 语法错
     - 根因：Service 内部 `offset=(page-1)*size`，caller 传 page=0 致负 offset
     - 修复：`findActive(0, 10000)` → `findActive(1, 10000)`（1-indexed 契约）
  2. **Bug #2**: `FeishuNotifyService` 手工拼 JSON + 错误信息未转义 → 飞书 400 code:9499
     - 根因：Python 多行 traceback + Windows 路径反斜杠 + Java `append("\n")` 真换行 拼进 JSON 字符串
     - 修复：错误信息截断 80 字 + 去换行 + `detail.append("\\n")` 字面 \n
- 真实验证结果：17:14:00 Scheduler 触发 → 飞书通知发送成功 → 飞书群收到卡片
- 118 tests 复跑零回归，cron 已回滚为 0 0 * * * ?，smoke.log + extracted.json 已清理
- 当前 git status：2 个 M（FeishuNotifyService + InspectionService bug fix），待 commit
- **教训**：(1) 分页 API 0/1 indexed 契约要明确；(2) 手工拼 JSON 是 anti-pattern，应用 Jackson；(3) **端到端冒烟能发现 unit test 抓不到的跨方言/跨网络问题**
