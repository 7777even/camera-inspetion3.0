### 发现的问题
- **P1-1**: trigger 端点同步阻塞 — `InspectionController.trigger()` 同步调用 `executeInspection()`（非 @Async），HTTP 响应需等全量巡检完成（~4min），curl 10min+ 未返回
- **P1-2**: `QueqiaoNotifyService.notifyNewData()` 未设置 `X-API-Key` 头，回调会被鹊桥 `ApiKeyAuthInterceptor` 返回 401
- **P2-1**: `smoke-test.sh` 参数名错误 — `start_date`/`end_date` 应为 `start`/`end`；`inspect_id` 应为 `inspectId`
- **P2-2**: `smoke-test.sh` notify 端点错误 — `GET /api/notify` 应为 `POST /api/notify/new-data` + `X-API-Key` 头 + JSON body
- **P2-3**: `enviro.queqiao.callback-url` 默认为空，自动同步仅靠 30min 定时任务

### MCP 工具参数名（重要，易错）
- `trigger_inspection`: reason
- `get_inspection_summary`: start, end（不是 start_date/end_date）
- `get_camera_status`: cameraName, historyDays
- `get_inspection_ledger`: date, status, enterprise
- `download_ledger_docx`: inspectId（不是 inspect_id）

### 鹊桥 notify 正确调用方式
- POST http://localhost:8081/api/notify/new-data
- Headers: X-API-Key: queqiao-notify-key-2026, Content-Type: application/json
- Body: {"syncVersion": N, "type": "inspection_completed"}
